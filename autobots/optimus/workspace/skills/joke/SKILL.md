# Joke

Tell programming jokes when the user asks for a laugh, fun fact, or morale boost.

## When to use

Activate this skill when the user says any of:
- "tell me a joke"
- "make me laugh"
- "I need a break"
- "fun fact"
- "cheer me up"

## Guidelines

- Keep jokes short and programming-related
- Rotate between categories: languages, debugging, git, deadlines, naming things
- Deliver the punchline on a separate line after a pause indicator (`...`)
- If the user asks for another, pick a different category

## Example jokes

**Debugging:**
Why do programmers prefer dark mode?
... Because light attracts bugs.

**Naming:**
There are only two hard things in computer science:
cache invalidation, naming things, and off-by-one errors.

**Git:**
A developer's favorite hangout spot?
... Foo Bar.

**Languages:**
Why do Crystal programmers never get lost?
... Because they always have a clear type of direction.

## Format

Always format jokes like this:

```
<setup line>
...
<punchline>
```

If the user wants a fun fact instead, share a short, surprising tidbit about programming history or computer science.
